# Plan Starstruck — étoiles légitimes

## Positionnement

`repoaudit` aide à préparer un dépôt open source : checks communautaires rapides, score lisible, zéro dépendance runtime.

## Seuils

| Niveau | Étoiles | Stratégie |
| --- | ---: | --- |
| Initial | 16 | Partages ciblés auprès de personnes déjà intéressées par l'outil |
| Bronze | 128 | Contenu utile (article, démo, talk court) + bouche-à-oreille |
| Argent | 512 | Adoption organique via CI snippets et mentions dans guides OSS |
| Or | 4096 | Produit durablement utile, docs excellentes, releases régulières |

## Textes prêts à publier (ne pas auto-poster)

### LinkedIn

> J'ai publié **repoaudit**, un CLI Python minimal pour auditer la maturité communautaire d'un dépôt GitHub local (README, licence, CONTRIBUTING, tests, Actions). Utile avant d'ouvrir un repo en open source. Feedback bienvenu : https://github.com/AVTAVANTTOUT2/Achievement

### Reddit (r/opensource / r/Python — respecter les règles du sub)

> Titre : repoaudit — CLI to check if a local GitHub repo is contributor-ready  
> Corps : short demo, link, ask for critique on check weights. No “please star”.

### Hacker News (Show HN)

> Show HN: repoaudit – audit local GitHub repo community readiness from the terminal  
> Focus on the problem and the zero-dependency design. Avoid engagement bait.

### X

> repoaudit: one command to score README / license / community files / tests / Actions on a local repo. Python, no runtime deps. https://github.com/AVTAVANTTOUT2/Achievement

### Discord / Slack

> Partage court + lien + invitation à tester sur un vrai repo. Pas de demande d'étoile.

## Règles éthiques

- Aucune ferme à étoiles, aucun échange artificiel, aucun faux compte
- Ne jamais demander de créer un compte uniquement pour étoiler
- Laisser l'utilité du projet convaincre

## Actions dépôt déjà prévues

- README professionnel
- Description et topics GitHub
- Release `v1.0.0` lorsque les tests passent
- Documentation claire (`docs/USAGE.md`)
