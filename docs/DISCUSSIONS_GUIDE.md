# Guide GitHub Discussions — Galaxy Brain

## Objectif

Obtenir des **réponses acceptées** utiles dans des discussions de type Q&A.  
Cela nécessite de véritables questions d'autres personnes (ou de contributeurs) et des réponses marquées comme acceptées par l'auteur ou un mainteneur.

## Règles

1. Ne créez pas de faux échanges entre comptes.
2. Ne marquez pas artificiellement vos propres réponses à vos propres questions uniquement pour le badge.
3. Répondez de façon précise, reproductible, avec commandes et contexte.
4. Une réponse acceptée doit résoudre réellement le problème posé.

## Bonnes pratiques pour poser une question

- Titre spécifique (pas « ça marche pas »)
- Version de Python / de `repoaudit`
- Commande exacte et sortie observée
- Arborescence minimale si pertinent
- Ce que vous avez déjà essayé

## Bonnes pratiques pour répondre

- Reproduire localement si possible
- Proposer une commande vérifiable
- Lien vers la doc (`docs/USAGE.md`) ou un test existant
- Distinguer cause / contournement / correctif durable
