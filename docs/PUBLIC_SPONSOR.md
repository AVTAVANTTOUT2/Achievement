# Public Sponsor — marche à suivre manuelle

Le badge **Public Sponsor** nécessite un **paiement réel** via GitHub Sponsors.  
Cette opération doit rester **manuelle**. Aucun paiement n'est automatisé ici.

## Choisir un bénéficiaire

1. Listez les projets open source que **vous utilisez réellement** (dépendances, outils, libs).
2. Vérifiez qu'un compte ou une organisation propose GitHub Sponsors.
3. Préférez un projet dont vous comprenez la valeur (maintenance, sécurité, docs).
4. Ne sélectionnez pas au hasard « pour le badge ».

## Contribution ponctuelle ou mensuelle

- **Ponctuelle** : remerciement clair pour un projet précis.
- **Mensuelle** : soutien durable si vous dépendez du projet au quotidien.
- Choisissez un montant confortable pour vous ; le badge ne vaut pas une dépense forcée.

## Rendre le sponsoring public

Pour l'achievement, le sponsoring doit être **public** (visibilité profil).  
Vérifiez l'option de visibilité dans l'interface GitHub Sponsors avant de confirmer.

## Marche à suivre (sans saisie de carte ici)

1. Ouvrir https://github.com/sponsors
2. Rechercher le développeur ou l'organisation éligible
3. Choisir un palier ponctuel ou mensuel
4. Vérifier que la contribution est **publique**
5. Finaliser le paiement **vous-même** dans l'UI GitHub
6. Conserver la confirmation pour votre suivi personnel

## État

`Action manuelle payante` — aucune carte bancaire n'est manipulée par l'automatisation de ce dépôt.
